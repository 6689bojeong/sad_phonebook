# PRD: 웹 기반 전화번호 관리 시스템 (Sad Phonebook)

## 1. 개요

### 제품 설명
로그인 없이 누구나 접근할 수 있는 심플한 웹 기반 전화번호부 관리 서비스.

### 목표
- 전화번호 정보를 빠르게 추가, 수정, 삭제할 수 있는 직관적인 UI 제공
- 별도 인증 없이 즉시 사용 가능한 단순한 접근성 확보

---

## 2. 기술 스택

| 구분 | 기술 |
|------|------|
| 프론트엔드 | HTML / CSS / JavaScript (또는 Next.js / React) |
| 백엔드 / DB | Supabase (PostgreSQL + REST API) |
| 호스팅 | Vercel |

---

## 3. 기능 요구사항

### 3.1 전화번호 목록 조회
- 저장된 모든 연락처를 목록 형태로 표시
- 각 항목에 이름, 전화번호, (선택) 메모 표시
- 이름 또는 번호로 검색 가능

### 3.2 전화번호 추가
- 이름과 전화번호를 입력하는 폼 제공
- 필수 입력값: 이름, 전화번호
- 선택 입력값: 메모
- 저장 버튼 클릭 시 Supabase에 즉시 반영

### 3.3 전화번호 수정
- 목록에서 특정 항목 선택 후 편집 가능
- 수정 완료 시 Supabase 데이터 업데이트

### 3.4 전화번호 삭제
- 목록에서 특정 항목 삭제 가능
- 삭제 전 확인 다이얼로그 표시
- 삭제 시 Supabase에서 해당 레코드 제거

---

## 4. 비기능 요구사항

| 항목 | 내용 |
|------|------|
| 인증 | 불필요 (로그인 없이 사용) |
| 접근성 | 모바일 / 데스크톱 반응형 UI |
| 성능 | 목록 로딩 1초 이내 |
| 배포 | Vercel을 통한 자동 배포 (GitHub 연동) |

---

## 5. 데이터 모델

### `contacts` 테이블 (Supabase)

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | uuid (PK) | 자동 생성 고유 ID |
| name | text (NOT NULL) | 연락처 이름 |
| phone | text (NOT NULL) | 전화번호 |
| memo | text | 메모 (선택) |
| created_at | timestamp | 생성 시각 (자동) |

---

## 6. 페이지 구성

### 메인 페이지 (`/`)
- 전체 연락처 목록
- 검색 입력창
- 새 연락처 추가 버튼
- 각 항목에 수정 / 삭제 버튼

---

## 7. 범위 외 (Out of Scope)

- 사용자 인증 / 권한 관리
- 연락처 그룹 / 카테고리 분류
- 연락처 가져오기 / 내보내기 (CSV 등)
- 중복 번호 경고
